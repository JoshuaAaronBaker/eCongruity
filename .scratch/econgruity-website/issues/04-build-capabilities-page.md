## What to build

Build the Capabilities page showing examples of the kinds of work eCongruity can provide. The page should include areas such as workflow automation, app development, enterprise CMS, learning management, mobile strategy, process improvement, organizational efficiencies, and innovation-as-a-service while making clear that these are examples, not packages or fixed offerings.

## Acceptance criteria

- [ ] The page presents Capabilities as examples of possible work, not as packages.
- [ ] The page includes the capability areas from the Figma copy and domain glossary.
- [ ] The page reinforces that every client receives a Tailored Solution.
- [ ] The page avoids package, tier, pricing, and service-menu language.
- [ ] Capability content helps an Organizational Leader recognize where their challenge might fit.
- [ ] The page includes a clear path to Approach and Contact.
