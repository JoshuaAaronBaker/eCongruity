Title: Establish Structured Site Content, Layout, and Navigation

## Parent

.scratch/econgruity-website/PRD.md

## What to build

Create the shared foundation for the multi-page eCongruity website. The site should behave like a traditional website with separate pages for Home, Approach, Capabilities, About, and Contact. Establish shared layout, navigation, footer, design tokens, and structured content conventions so later page slices can plug into a consistent experience. Testimonials should be structured content used as proof on the Home page, not a standalone page.

## Acceptance criteria

- [ ] The site has routes for Home, Approach, Capabilities, About, and Contact.
- [ ] Shared navigation links to each top-level page and clearly identifies eCongruity.
- [ ] Shared footer includes the core positioning line and relevant business/contact placeholders from available source material.
- [ ] Brand colors, typography choices, spacing, and common layout primitives are centralized enough for later pages to reuse.
- [ ] Structured content conventions exist for capabilities, testimonials, and founder information.
- [ ] The site builds successfully with the existing Astro/Tailwind stack.

None - can start immediately
