Title: Establish Structured Site Content, Layout, and Navigation

## Parent

.scratch/econgruity-website/PRD.md

## What to build

Create the shared foundation for the multi-page eCongruity website. The site should behave like a traditional website with separate pages for Home, Approach, Capabilities, About, and Contact. Establish shared layout, navigation, footer, design tokens, and structured content conventions so later page slices can plug into a consistent experience. Testimonials should be structured content used as proof on the Home page, not a standalone page.

## Acceptance criteria

- [x] The site has routes for Home, Approach, Capabilities, About, and Contact.
- [x] Shared navigation links to each top-level page and clearly identifies eCongruity.
- [x] Shared footer includes the core positioning line and relevant business/contact placeholders from available source material.
- [x] Brand colors, typography choices, spacing, and common layout primitives are centralized enough for later pages to reuse.
- [x] Structured content conventions exist for capabilities, testimonials, and founder information.
- [x] The site builds successfully with the existing Astro/Tailwind stack.
